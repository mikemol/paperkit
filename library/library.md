# The Paperkit Concept Library

*Each concept's witness is authored and GRADED here once; every view imports the certificate (verdict + engine fingerprint) rather than re-authoring a parallel proof.*

## Adequacy: The Δ Falsifiability Concept

The adequacy witness runs the real grader over a fixture, so mutating an engine def-site flips it — the certificate's sensitivity fingerprint IS the engine, which every importing view inherits [@adequacy-pitch]; the same witness under the deep key certifies the grade ladder — a presupposed file grades vacuous, a content-sensitive command behavioral — so the paper's deep exposition imports the identical proof the pitch does [@grade-ladder].

## Model: A Claim Is a Record

A claim is one bibliography entry — a statement, its rubric section, its dependencies, and its check — parsed by the engine's OWN bib reader, so mutating a parser def-site flips the witness and the certificate's fingerprint IS that parser [@rm-model]; the same witness under the paper's key certifies that a claim is a single record — the deep view imports the identical proof the README pitch does [@claim-is-record].

## Resolver: The Verifier and the Escape Hatch

Each claim carries a machine-checkable verifier — the check field names a verb and the gate RESOLVES it; the parser and the resolver are engine, so the certificate's fingerprint is them [@rm-verifier]; the same witness under the paper's key certifies that each claim bears a machine-checkable verifier [@claim-bears-check]; cmd is the escape hatch every check reduces to, and a new domain adds a verifier in config without touching the engine — a config-supplied type resolves, an unregistered one does not [@rm-resolver-cmd]; the paper's cmd-escape face imports this same certificate — cmd is the universal escape hatch every other check reduces to [@cmd-escape]; and the paper's new-domain-adds face too — supporting a new domain means adding verifiers, not editing the engine [@new-domain-adds]; the resolver component's certificate — a verifier is named type:target, EVERY declared verb dispatches to a real branch (read from resolver.VERBS, nothing re-listed), the built-in set is closed, and a custom type dispatches through the registry [@rm-resolver]; the paper's verifier-named face imports this certificate — the type prefix selects the verb [@verifier-named]; and its gate-dispatches face — the gate dispatches through a small registry of check types, built-in and custom [@gate-dispatches]; and its two-builtins face — a verifier ships built in per verb, and the set is pinned ENUMERATIVELY rather than by count, closing the adequacy gap that let a fifth verb falsify the prose silently [@two-builtins].

## Gate: Enforcing the Invariants

An unverified sentence cannot ship — a claim whose check FAILS blocks the gate; the gate is engine, so its def-sites are the fingerprint [@rm-noship]; the paper's fail-omits face imports this same certificate — a claim whose verifier fails simply does not appear in the prose [@fail-omits]; the gate ENFORCES its invariants — the committed prose equals its projection, and every cited claim's check passes — so violating each makes it RED [@rm-cmds-inv]; the paper's self-host face imports this same certificate — the gate that accepts the paper is the very subject it describes, enforcing the invariants the paper states [@gate-is-subject].

## Projector: The Document Is the Projection

The projector component's certificate — a document IS the projection of its claim-DAG: the title, every rubric section, and every claim's prose appear in the emitted document, which leads with a heading [@rm-pitch]; the paper's thesis face imports this certificate — a scholarly paper as the projection of a claim-DAG [@paper-is-projection]; and its engine face — the projector emits the whole document from the warrant set, populated sections and empty ones alike [@projector-emits]; two commands do the work — PROJECT makes the document and GATE verifies it: the exact projection passes, hand-edited drift is rejected [@rm-cmds]; the paper's drift face imports this same certificate — the gate rejects any prose that has drifted from its projection [@gate-rejects-drift].
